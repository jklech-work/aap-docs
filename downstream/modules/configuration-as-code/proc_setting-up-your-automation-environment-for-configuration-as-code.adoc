= Setting up your automation environment for Configuration-as-Code

Configuration-as-Code is a way of working where you define and manage the configuration of the Ansible Automation Platform (AAP) itself using the version-controlled configuration files (YAML, JSON), instead of clicking through the web UI.

As an Ansible content developer, you can use the Configuration-as-Code approach to apply settings on your AAP Controller to get the following benefits:


.Prerequisites

* You have a GitHub account.
* You have a link:https://quay.io[Quay.io] account.
* You have functional AAP and attached the relevant RHEL subscriptions on all of its components.
* Your AAP Gateway instance is accessible.


.Procedure

. Fork, then clone to your local machine this link:https://github.com/rohitthakur2590/aap-2.6-ansible-platform-testathon-playbooks[Github repository].


. Update the `~/aap-2.6-ansible-platform-testathon-playbooks/vars/aap.yml` file with the following variables:
+
[source,yaml,subs="+quotes"]
....
---
aap_hostname: "http://<YOUR_GATEWAY_IP>"
aap_username: "<YOUR_GATEWAY_USER>"
aap_password: "<YOUR_GATEWAY_PWD>"
aap_validate_certs: false
....


. Update the `~/aap-2.6-ansible-platform-testathon-playbooks/vars/user.yml` file with the following variable:
+
[source,yaml,subs="+quotes"]
....
---
rh_username: "<YOUR_OFFICIAL_RH_USERNAME>"
....


. On your local machine, commit and push all the updated variables to your forked GitHub repository so that AAP Controller can later read in the correct data:
+
[subs="+quotes"]
....
git add .
git commit -m "Update variables for your_official_rh_username"
git push origin <relevant_branch_name>
....


. Through the AAP Gateway UI, register and save an Execution Environment with the following values:
+
* Name: ansible-platform-ee
* Image: quay.io/rothakur18/platform-ee02
* Pull: Always pull container before running
* Description: Execution Environment to test the CaC capability
* Organization: Default
+
image::cac-create-ee.png[Create Execution Environment]


. Through the AAP Gateway UI, create a new Project with the following values:
+
* Name: Platform collection testing
* Description: Automation resources to test the CaC capability of RBAC modules from ansible.platform collection
* Execution Environment: ansible-platform-ee
* Organization: Default
* Source Control Type: Git
* Source Control URL: https://github.com/<YOUR-GITHUB-USERNAME>/aap-2.6-ansible-platform-testathon-playbooks
+
image::cac-create-project.png[Create Project]


. Through the AAP Gateway UI, create a job Template with the following values:
+
* Name: 1487-1-1-TC-RDEF
* Description: Create a role definition with permissions
* Job type: Run
* Inventory: Demo Inventory
* Project: Platform collection testing
* Execution Environment: ansible-platform-ee
* Playbook: 1487-1-1-TC-RDEF.yaml
+
image::cac-create-job-template.png[Create job Template]


. If needed, create job templates for the other playbooks in the configured GitHub repository.
